### 1. 【学术前沿】MIRIX 六模块记忆架构：复杂 Agent 的记忆新范式

**来源**: [SegmentFault 2026-01-07](https://segmentfault.com/a/1190000047526306)

加州大学圣迭戈分校提出的 **MIRIX 架构**，将 Agent 记忆按功能拆分为六个独立模块，代表了当前最精细的记忆功能划分思路：

| 模块 | 定义 | 示例 |
|------|------|------|
| 核心记忆 | 用户基础信息与核心偏好 | 过敏史、职业 |
| 情景记忆 | 按时间排序的事件记录 | "2025年10月咨询过巴黎旅行" |
| 语义记忆 | 概念性知识与事实 | "奥赛博物馆以印象派藏品闻名" |
| 程序记忆 | 标准化操作流程 | "订单退款的三步流程" |
| 资源记忆 | 多模态工作资料 | 文档、截图、音频片段 |
| 知识金库 | 加密存储的敏感信息 | 银行卡号、隐私数据 |

MIRIX 采用"元管理者+专项管理者"架构：元记忆管理者负责任务分发与协调，各专项管理者并行处理对应模块，汇总后反馈给 Agent。

> **钻头评**: MIRIX 的最大价值在于将"记忆"从统一存储拆解为功能导向的独立组件。这就像人脑有海马体负责情景记忆、有新皮层负责语义知识一样——专业化分工才能支撑复杂 Agent 的高效运作。

### 2. 【技术架构】MemGPT 分层内存：突破"无限上下文"的新路径

**来源**: [CSDN 2026-03-06](https://blog.csdn.net/2401_85379281/article/details/156564382)、[CSDN 2026-03-02](https://blog.csdn.net/CSDN_430422/article/details/156614388)

MemGPT 提出**分层内存架构**，受操作系统虚拟内存启发，将 Agent 记忆映射为：

- **主上下文（Main Context）**≈ LLM 原生上下文窗口，存储实时交互数据，支持毫秒级访问
- **外部上下文（External Context）**≈ 归档存储 + 调用存储，存储非实时数据
- **内存管理器**≈ 由 LLM 自主控制，通过函数调用实现数据在主/外部上下文间的"分页加载/存储"

这一架构的核心创新在于：**让 LLM 自己决定什么时候"记住"、什么时候"遗忘"**，模拟了人类内存管理的自主性。

> **钻头评**: MemGPT 的本质是将操作系统"虚拟内存分页"的思想引入 AI——不是无限扩大物理内存，而是用智能调度让有限窗口发挥无限效果。这可能是未来 Agent 记忆系统的主流方向。

### 3. 【工程实践】上下文工程三大策略：缩减 / 卸载 / 隔离

**来源**: [CSDN 2026-03-06](https://blog.csdn.net/2401_85379281/article/details/156564382)

短期记忆直接参与 Agent 交互，随对话增长必然面临 Token 超限。业界已形成三种成熟策略：

**① 缩减（Reduction）— 有损压缩**
- 摘要：LLM 对内容进行概括，丢弃次要细节
- 预览：仅保留前 N 字符作为提示
- *适用*：历史消息、冗长讨论

**② 卸载（Offloading）— 无损存储**
- 将完整内容移至外部存储（文件/数据库）
- 上下文仅保留引用指针（UUID/文件路径）
- *适用*：网页搜索结果、超长工具输出、临时计划

**③ 隔离（Isolation）— 模块化拆分**
- 多 Agent 架构下，不同子 Agent 使用独立上下文
- 避免信息交叉污染，降低单 Agent 上下文负载
- *适用*：任务指令清晰、仅关注最终输出的场景

> **钻头评**: 三种策略并非互斥，而是组合使用。核心原则是：近期优先保留、数据类型分级、信息可恢复性决定处理方式。

### 6. 【行业动态】Agentic RAG 成为 2025 Agent 落地核心模式

**来源**: [腾讯网 2025-08-14](https://new.qq.com/rain/a/20250814A084VO00)

2025 年 AI Agent 六大趋势中，**Agentic RAG** 位列第一：

- **核心定义**：在传统 RAG 基础上引入 Agent 机制，让检索与生成过程更自主、更可控、具备多轮决策能力
- **代表公司**：Perplexity、Harvey AI、Glean AI
- **应用场景**：不止于单一领域，医疗健康等行业同样具备潜力
- **关键差异**：传统 RAG 是"一次检索"，Agentic RAG 是"检索→判断→再检索→生成"的循环决策

> **钻头评**: Agentic RAG 的本质是将 RAG 从被动工具升级为主动 Agent——它不只是找资料，而是会判断"找的对不对"、"够不够"、"还需要什么"。记忆系统在其中扮演的角色：从被检索的"静态知识库"演变为"动态交互记忆体"。
