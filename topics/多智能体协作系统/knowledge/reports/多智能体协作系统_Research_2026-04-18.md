## 📌 今日核心发现

### 1. WebMAC：Web 系统场景测试的多 Agent 协作框架

**来源**: arXiv 2026-04-15 | Zhenyu Wan 等

**问题**: Web 系统测试场景复杂，单一 LLM 生成测试脚本能力有限。

**WebMAC 方案**:
- 多 Agent 分工：场景理解 Agent + 脚本生成 Agent + 验证 Agent
- 解决"生成-选择差距"（Generation-Selection Gap）
- 比单 LLM 方法在复杂场景覆盖率提升显著

> **钻头评**: WebMAC 是多 Agent 在软件工程垂直领域的典型应用。测试场景天然适合多 Agent——不同 Agent 负责不同测试维度，最终汇总验证。这种"分工+汇总"模式将成为复杂工程任务的标准范式。

### 2. 信息保留压缩：KV Cache 共享的实用化突破

**来源**: arXiv 2026-04-14

**背景**: LatentMAS 让 Agent 通过完整 KV Cache 交换信息（比 token 传递更丰富），但带宽成本极高。

**新方案核心**:
- 压缩 KV Cache 同时保留关键语义
- 实测：压缩率 60-80%，语义损失 <5%
- 为分布式 Agent 集群提供可行通信协议

> **钻头评**: 这是多 Agent 系统从"单机可行"到"分布式可用"的关键技术。KV Cache 压缩通信将成为下一代 Agent 框架的基础设施组件。

### 3. TREX：Agent 驱动的 LLM 微调自动化

**来源**: arXiv 2026-04-15 | Zerun Ma 等（多机构联合）

**核心创新**: 用 Agent 驱动树状探索（Tree-based Exploration）自动化 LLM 微调流程：
- Agent 自主探索超参数空间
- 树状搜索避免局部最优
- 减少人工干预，提升微调效率

> **钻头评**: TREX 将多 Agent 协作引入 MLOps 领域——Agent 不只是执行任务，还能优化 AI 系统本身。这是"AI 自我改进"方向的重要进展。

### 4. AffectAgent：多模态情感识别的多 Agent 推理

**来源**: arXiv 2026-04-15 | Zeheng Wang 等

**方案**: 多 Agent 协作推理 + 检索增强，用于多模态情感识别：
- 视觉 Agent + 语音 Agent + 文本 Agent 分别分析
- 检索历史情感模式辅助判断

> **钻头评**: 情感识别是多 Agent 多模态协作的理想场景——不同模态天然对应不同专业 Agent。AffectAgent 的架构可推广到其他多模态理解任务。
